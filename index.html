回调队列callback queue

上面的例子，当setTimeout执行结束的时候，是不是就应该回到执行栈，进行执行输出呢？

答案：并不是！

此时，倒计时结束后的setTimeout的可执行函数，被放如了回调队列

最后，setTimeout的可执行函数，被从回调队列中取出，再此放入了执行栈

这样的执行过程就叫 event loop事件循环

一，前端漏洞的介绍

前端漏洞包括:

1，前端sql注入

2，xss（cross site scripting）跨站脚本攻击

3，csrf (Cross Site Request Forgery) 跨站请求伪造

前端的攻击方式种类繁多，初次接触，不太容易理解这些词汇，所以我们从危害开始说起。

二，前端漏洞的危害

4，万幸新浪微博反应迅速，1小时后漏洞被修复，xss攻击没有大范围扩散

2，xss攻击的发生

xss被称为跨站脚本攻击

2.1 同源策略

xss的核心就是跨站两个字，为什么要跨站呢？因为浏览器存在同源策略

同源策略规定：不同域的客户端脚本在没有明确授权的情况下，不能读写对方的资源

简单来说就是客户端js不能够跨域(协议、端口、域名)请求资源

2.2 xss跨站脚本攻击

恶意攻击者他们的目的是将自己的脚本注入到客户的浏览器中运行，同时攻击者想要做的肯定是一系列事情，所以不是几句代码就能解决，所以他们更希望的是能够把脚本代码文件，通过引入的形式加载进浏览器。

这样做有两个好处：

1，攻击者维护攻击代码方便，可读性强
2，能够绕过浏览器的输入限制。

而这么做的后果，就是不可避免的要面对浏览器的同源策略，所以就要想办法绕过同源策略
