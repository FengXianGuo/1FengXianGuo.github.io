三，任意值

例：
let num:number = 7;
num = "str";

// index.ts(2,1): error TS2322: Type 'number' is not assignable to type 'string'.

定义好类型的变量，修改时会报错

🐽：any则可以赋值为任意值

🐽：any可以简单理解成熟悉的若类型变量

四,未声明类型的变量

例：
let res;
res = 7;

等价于:

let res:any;
res = 7;

五，类型推论

ts会默认进行类型推论

例：
let res = "str";
res = 7;

等价于：

let res:string = 'str';
res = 7;

既是说：
变量声明，未赋值，默认为any；
变量声明，已赋值，则为逆推出来的类型（string）；

六，联合类型

例:
let res : string | number;
res = 7;
res = "seven";

原则：尽量约束，any毕竟有风险；

所以就有了联合类型，使用|来兼容多个类型。

七，联合类型引出的一些问题

联合类型的属性问题：
例：
function getLength(data:string|number){
    return data.length;
}

上述代码，编译会报错：
因为：number类型没有length属性；

例:
let res:string|number;
res = "seven";
console.log(res.length);
res = 7;
console.log(res.length);

上述代码，同样会报错：
因为：number类型没有length属性；

对象的类型-接口

一，接口的概念

需要明确

暂定：
1，接口用于约束对象，
    相当于规定对象的形状
2，接口是类的一部分行为的抽象

二，完全约束接口

例：
interface Man {
    name:string,
    age:number
}
let boy:Man = {
    name:"boy",
    age:12
}

interface：
用于定义一个接口

上述代码的意思是：

interface定义了一个接口man，
其中约束了name和age的类型。

let创建变量boy，类型是man，
规定了boy的形状必须与man一致。

🐽：boy的属性不能少，也不能多，否则都报错。

🐽：应用场景：
提交后端数据前，约束格式。

此时接口属性固定为2个

三，不完全约束接口

前面严格约束了一个接口
但当有些属性可有可无的情况，就不太适用了

例：
interface Man {
    name:string,
    age?:number
}
let boy:Man = {
    name:'boy'
}

上述代码中多了个？
？号的意义是，规定age可填可不填

🐽：boy的属性除了age以外，还是不能少，也不能多，否则报错。

此时接口属性固定有1||2个

四，添加任意属性

严格约束接口
不完全约束接口

上面这两种属性都是固定的。

所以需要一个添加任意属性的方法；

例：
interface Man {
    name:string,
    age?:number,
    [name:string]:any
}
let boy:Man = {
    name:'boy',
    key1:"value1",
    key2:"value2"
}

上述代码中，使用[中括号包裹的变量]，即为任意属性名，any为任意属性值；

五，添加任意属性带来的问题

因为任意属性的权重>可选属性|确定属性
是个包裹关系

例：
interface Man {
    name:string,
    age?:number,
    [name:string]:string
}
let boy:Man = {
    name:'boy',
    key1:"value1",
    key2:"value2"
}

上述代码会报错
因为任意属性的类型是：string
而可选属性age的类型是：number

string不包含number所以报错

六，只读属性

这里的只读有些歧义。

其实可以简单理解为：

类似const定义常量。

例：
interface Man {
    readonly age:number
}

let boy : Man = {
    age:12
}

上述代码中多了一个readonly
readonly：
定义的属性只可定义一次。

此时：
boy.age = 13；
则会编译报错；

数组

一 类型+方括号，来表示数组

例：

let arr : number[] = [1,2,3,4,5];

其中
：用于指定变量类型
number 是数字类型
[] 中括号是指数组

总结：arr规范了是数组，切里面每一个元素都是数字类型

如果要包含联合类型：

let arr : (number|string)[] = [1,2,3,"4",5];
🐽：（number|string）有中括号包裹

二 泛型来表示数组

这里浅尝截止，不做深入

let arr : Array<number|string> = [1,2,3,"4",5];

上面类型直接就是数组这个类
然后通过<定义内部约束>

三 以接口形式表示数组

interface NumberArray {
  [index: number]: number;
}
let fibonacci: NumberArray = [1, 1, 2, 3, 5];

上述为用接口约束数组的方式，
index为下标，
规定，下标为key，value为number类型

四，数组的常用方法

let arr : any[] = [1,2,3,"4",5];

就是使用any去定义数组

五，类数组

function sum() {
  let args: IArguments = arguments;
}

arguments不是数组，如果用数组去约束，会报错

其实ts中已经定义好了内置对象，用来约束这一类

⚠️：重点关注IArguments这些内置对象

函数

一 约束函数

例：

let mySum = function (x: number, y: number): number {
  return x + y;
};

上面的例子有缺陷

= 右侧，规范了入参x，y都是number，并且规定了返回值为number

= 左侧，无规范

let mySum: (x: number, y: number) => number = function (x: number, y: number): number {
  return x + y;
};

正确的做法，应该是如上

let mySum：开始定义mySum的类型
也就是（x:number,y:number)=>number这里
= 右侧才是真正的函数体


简单尝试：let mySum = (x: number, y: number):number => x+y; 

上面的例子，函数的参数是固定的2个

二 通过接口约束

interface Fn {
    (x:number,y:number):boolean;
}
let fn : Fn = function(x:number,y:number){
    return x+y>1
}

上面就是通过接口定义的函数形状

上面的例子，函数的参数是固定的2个

三 可选参数

前面的函数，参数都是2个，多一个少一个，都会报错

例：
function add(x:number,y?:number){
    return x+y;
}
add(1,2);

上面就是一个添加了可选参数的例子

🐽：可选参数必须在固定参数后面，否则报错


此时参数个数是1||2，还是固定的

四，默认值

function add(x:number=1,y:number){
    return x+y;
}
add(undefined,2);

上面就是结合es6默认值的应用

五，剩余参数 || 多个参数

前面我们处理的都是固定参数的情况

例：
function add(array:any[],...items:any[]){
    items.forEach(function(item){
        array.push(item)
    })
    return array
}
let arr :any[] = [];

add(arr,1,2,3,4,5)

上面主要是使用es6的rest参数来获取

六，重载

例：
function reverse(x: number | string): number | string {
  if (typeof x === 'number') {
    return Number(x.toString().split('').reverse().join(''));
  } else if (typeof x === 'string') {
    return x.split('').reverse().join('');
  }
}
上述代码的功能是反转，输入数字反转，输入字符串也能够反转

存在问题：
function reverse(x: number | string): number | string
单独看上面这句话的话，很难准确判断究竟类型检查的目的：
看起来：输入字符串||数字，可以返回数字||字符串

但其实，我们想要的是：
输入数字，返回数字
输入字符串，返回字符串

所以上面的写法存在不明确的地方

解决方案：通过重载

function reverse(x: number): number;
function reverse(x: string): string;
function reverse(x: number | string): number | string {
  if (typeof x === 'number') {
    return Number(x.toString().split('').reverse().join(''));
  } else if (typeof x === 'string') {
    return x.split('').reverse().join('');
  }
}

前两次定义就是为了精确定义关系。

类型断言

ts默认会执行类型推论，会推测一个变量是什么类型；

而类型断言的意思，就是告诉ts，不用推了，就这个类型，肯定没错。

例如:
function test(x:number|string):number{
    return x.length;
}

上面的例子，会报错

🐽：number没有length属性

前面我们了解过，联合类型只能访问公共方法；

但length不是公共方法，理所当然报错了。

那有没有什么方法来解决呢？

方法一：

function test(x:number|string):number{
    return (x as string).length;
}

上面的方法是通过“值 as 类型”的方法来实现的。

强行，将x指定为string类型；

方法二：
function test(x:number|string):number{
    return (<string>x).length;
}

上面的方法是通过“(<类型>值)”的方法来实现的。

🐽：不允许断言成一个联合类型中没有的类型。

#声明文件

一，声明

如果我们想要使用jquery，去获取一个元素

jQuery(".main")

这样ts会报错。
因为ts并不认识jQuery是什么。

这种情况，就需要告诉ts，jQuery是什么。

declare var jQuery: (string) => any;

jQuery('.main');

其中，declare是关键字，只会出现在编译阶段，用于定义；

二，引用

ts中声明类型，一般放到一个单独的文件夹中

例：
// jQuery.d.ts
declare var jQuery: (string) => any;


然后在需要的文件中，通过///三道斜线来引用
/// <reference path="./jQuery.d.ts" />

jQuery('#foo');

三，第三方声明文件

像这些定义，ts其实已经帮我们定义好了

可以在下面这个网站搜索：
http://microsoft.github.io/TypeSearch/

搜索到后，安装即可
例如：
npm install @types/jquery --save-dev
