使用webpack重构koa2多页应用

发表于 2017-08-08
字数统计 2,735 | 阅读时长 13

一，项目背景

老的技术栈：

koa2
gulp
reguirejs
ejs
artTemplete

环境：

node：4.4.5
npm:2.15.0

背景：

1，使用koa2搭建中间层，负责处理视图层的展示；
2，使用ejs，和arTemplete来处理ssr(服务端渲染)和csr(客户端渲染)；
3，使用gulp打包静态资源。


二，存在问题

1，node -v4.4.5，版本过低，部分新api和新时代插件不支持。
2，requirejs没有添加babel支持。

总结：团队成员迫切需要切换es6


三，为什么选择webpack

其实，支持es6的方案有好多，例如：

1，gulp-babel
2，babel
3，ts
4，webpack

比较前面3种方案

gulp-babel：虽然成熟，但没什么新鲜感；
babel：专门用来编译es6，没有其他惊喜；
ts：对babel支持很好，但不知道是能否成为规范，需要学习成本。

之所以选择webpack：

1，强大的打包功能，能够把所有的资源全部打包在一起。
2，对babel的支持。
3，社区活跃。
4，增长技术积累。

其中强大的打包功能是最吸引人的。

想必大家都遇到过这样的场景：

1，A工程师完成了一个功能在Aapp上，
2，产品突然想在Bapp上实现同样的功能，
3，B工程师找到A工程师，拷贝了A的代码，
4，B拿到了(1)html、(2)js、(3)css，
5，产品又想改需求了...
6，A工程师改
7，B工程师改
8，产品又想改需求...
9，A改
10，B改
11,...
12,...
13,...

针对上面这种情况：
首先肯定是将功能模块封装起来，由A，B工程师同时引用同一个功能模块。

但是
html，
js，
css
本身就是三个部分，怎么能封装到一起？

努力思考一下：

html可以通过字符串的形式，插入到js当中。(react的原理，差不多也是如此)

这样的话，需要封装的内容就变成了2部分：
js，
css

但css怎么和js合体呢?

这就是webpack的强大之处了，webpack可以通过各种loader，把各种资源都打包成js，拿到我们的问题上来，webpack把css打包成了js。

最终:
我们只需要暴露js即可。


四，在express中引入webpack

关于webpack的使用，网上有很多教程，不过大部分都是为单页应用服务，很少与express或者koa结合。
尤其是关于webpack-dev-server的部分更是会误导多页应用开发人员的视线。

webpack的基本知识，这里不再赘述。

express中使用的webpack中间件是：

webpack-dev-middle

⚠️注意：这个中间件不能够直接在koa2中使用。
因为koa2中没有显式的req和res，而是变成了ctx上下文，所以很多express的插件都不能够直接使用。

使用webpack中间件

node服务中，重中之重就是中间件，想要在node中使用webpack就需要依赖webpack的中间件。

核心概念：当请求访问服务器时，同步打包并编译文件，生成编译后的新文件，响应到新的文件。


五，在koa2中引用webpack

搜索webpack官网中的webpack-dev-middle，可以看到如下：

Note: The webpack-dev-middleware is for advanced users. See webpack-dev-server for a ready-to-use solution.
Note: For multiple webpack configuration see middware or dev-middleware
If you are looking for this middleware for koa2 ecosystem loaded with HMR support see at koa-webpack-middleware.

官网说：想要在koa2中使用，可以去查看koa-webpack-middleware插件.

但是，测试之后，发现报错，不能正常使用

接下来，又找了数个插件:
koa-webpack-dev-middleware:失败
webpack-koa2-middleware:成功

不知道为什么官网提供的并不好用，反而是一个国内的coder包装过的能用，这有可能跟我的版本有关。

十，注意事项

1,webpack版本问题：

由于webpack发展迅速，现在已经发展到了webpack3.4.1;
其本身已经使用了箭头函数语法()=>{};
但我的项目node版本是v4.4.2，本身不支持es6，所以后来降级到了webpack3.3.0.
